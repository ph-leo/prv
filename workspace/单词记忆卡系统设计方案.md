# 单词记忆卡系统设计方案

## 一、系统概述

### 1.1 系统名称
**WordCard - 智能单词记忆卡系统**

### 1.2 核心功能
- 上传单词图片（词汇表/单词书照片）
- AI自动识别提取单词
- 生成互动式记忆卡片
- 支持在线练习和复习

### 1.3 目标用户
- 高中生（高考词汇备考）
- 大学生（四六级/考研）
- 英语学习者

### 2.1 技术栈

## 三、核心功能模块

### 3.1 图片上传与识别模块

#### 功能流程
```
1. 用户上传图片 → 2. 图片预处理 → 3. OCR识别 → 4. 结构化提取 → 5. 生成单词列表
```

#### 支持的图片类型
- 词汇表照片（手机拍摄）
- 单词书扫描件
- Excel/Word截图

#### 识别字段
| 字段 | 说明 | 示例 |
|------|------|------|
| 序号 | 单词编号 | 1, 2, 3... |
| 单词 | 英文单词 | ability |
| 音标 | 国际音标 | /əˈbɪləti/ |
| 词性 | 单词词性 | n. |
| 释义 | 中文释义 | 能力，才能 |
| 难度 | 难度等级 | ★★☆☆☆ |
| 课本 | 所属教材 | 必修一 |
| 单元 | 所属单元 | Unit 1 |

### 3.3 练习模式模块

#### 模式1：自由练习
- 自主选择单词范围
- 随机抽取卡片练习
- 随时查看答案

#### 模式2：顺序学习
- 按课本/单元顺序学习
- 新词学习 + 复习
- 进度保存

#### 模式3：智能复习（艾宾浩斯）
```
复习周期：20分钟 → 1小时 → 9小时 → 1天 → 2天 → 6天 → 31天
```

#### 模式4：挑战模式
- 限时答题
- 连续答对奖励
- 排行榜

- 学习进度饼图

## 五、技术实现要点

### 5.1 OCR识别流程
```python
# 1. 图片上传
# 2. 图片预处理（旋转校正、去噪）
# 3. 表格检测（识别表格结构）
# 4. OCR文字识别（PaddleOCR）
# 5. 结构化提取（正则匹配字段）
# 6. 数据清洗（去重、纠错）
# 7. 存入数据库
```

### 5.2 艾宾浩斯复习算法
```python
def calculate_next_review(review_count):
    intervals = [20/60, 1, 9, 24, 48, 144, 744]  # 小时
    if review_count < len(intervals):
        hours = intervals[review_count]
    else:
        hours = 744  # 31天
    return now + timedelta(hours=hours)
```

### 5.3 卡片翻转动画
```css
.card {
    transition: transform 0.6s;
    transform-style: preserve-3d;
}
.card.flipped {
    transform: rotateY(180deg);
}
```
