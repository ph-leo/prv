## 🎯 检查流程

### 1️⃣ 每小时检查节点

| 时间点 | 检查内容 | 操作 |
|--------|----------|------|
| **00:00** | 0-1小时进度 | 检查报告更新 |
| **01:00** | 1-2小时进度 | 检查报告更新 |

| **...** | ... | ... |
| **23:00** | 23-24小时进度 | 检查报告更新 |

### 2️⃣ 检查内容清单

#### ✅ 基础检查（每次检查）
- [ ] 进度报告是否更新到最新时间点
- [ ] 完成病例数是否与上次一致或增加
- [ ] 是否有未处理的问题
- [ ] AI 服务状态是否正常

#### 📈 进度检查
- [ ] 本小时完成病例数是否合理
- [ ] 推理速度是否异常（过快/过慢）
- [ ] 完成率是否与预期一致

#### ⚠️ 问题检查
- [ ] 是否有新问题记录
- [ ] 问题处理状态是否更新
- [ ] 是否需要上报给负责人

### 3️⃣ 检查结果处理

| 情况 | 处理方式 |
|------|----------|
| **进度正常更新** | 记录状态，继续监控 |
| **进度未更新** | 立即询问测试工程师原因 |
| **进度异常** | 分析原因，可能需要调整方案 |
| **重大问题** | 询问测试工程师并上报负责人 |
| **多种异常** | 启动紧急响应流程 |

## 📞 问题上报流程

### 第一级: 工程师协助

当测试工程师遇到问题时：
1. 记录问题详情
2. 尝试解决
3. 更新进度报告
4. 如果 **30分钟内无法解决** → 报告我

### 第二级: 我的协助

当收到测试工程师求助时：
1. 分析问题类型
2. 提供解决方案
3. 如果 **1小时内外无法解决** → 报告负责人

### 第三级: 负责人介入

当问题严重时：
1. 提供完整问题描述
2. 已尝试的解决方法
3. 需要的资源请求
4. 等待负责人指示

## 📋 异常情况处理

### 情况1: 进度报告未更新

```bash
# 我的检查脚本
ssh -p 12222 D@127.0.0.1 "
cd /e/ai_test_MedGemma/ai/other &&
ls -lt test_results/v7.4_300cases/ | head -5 &&
echo '=== 最后更新时间 ===' &&
stat test_results/v7.4_300cases/summary_report_*.md 2>/dev/null | grep Modify
"
```

**如果超过 1 小时未更新**:
```bash
# 自动发送检查消息
echo "【自动检查】测试进度超过1小时未更新，请更新进度报告！"
```

### 情况2: AI 服务异常

```bash
# 检查 AI 服务
curl -s http://localhost:8000/health
```

**如果服务异常**:
- 检查 medgemma_api_plus.py 是否运行
- 尝试重启服务
- 如果无法恢复，记录并上报

### 情况3: 磁盘空间不足

```bash
# 检查磁盘空间
df -h /e
```

**如果空间不足**:
- 清理转换图像目录
- 检查是否有大文件
- 必要时暂停测试
